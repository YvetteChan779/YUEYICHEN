PLAN-B: Multimodal Perception & Closed-Loop Grasping for Humanoid Robot

Yueyi Chen · Algorithm Intern Project · Jetson AGX Orin · 2026-03 → 2026-05

planb-robot is a multimodal human–robot interaction system deployed on Jetson AGX Orin, covering the full closed loop of Speech → Semantics → Perception → Planning → Execution → Verification. This page documents the system architecture, key technical innovations, and quantitative results across five subsystems.

System Overview

graph LR
    A["🎤 Mic 16kHz"] --> B["VadQualityGate\n3-tier VAD"]
    B --> C["Streaming ASR\n4-engine Router"]
    C --> D["NLU Parser\nRegex + LLM"]
    D --> E["VLM + SAM3\nGrounding & Segmentation"]
    E --> F["Pipeline FSM\n8-state Machine"]
    F --> G["Navigation\n+ Grasping"]
    G --> H["Verify\ngrasp_check 3D"]
    H -->|"retry ≤ 3"| F
    H -->|"success"| I["✅ TTS\nFeedback"]

The system is organized as a linear pipeline where each stage's output serves as the next stage's input contract. Key design principles: (1) TTS reverse-gates VAD via /tts/playback_active to prevent self-triggering; (2) the verification layer feeds failures back to the planning layer (not idle), enabling retry on “executor reports success but vision reports failure”; (3) any stage's error can be grepped via ROS topic constants.

Streaming ASR Engine Router

Four ASR engines switchable within the Router: cloud_qwen3 (default, real-time streaming), cloud_paraformer (hot-swap backup), local Qwen3-ASR TRT (offline), and FunASR (chunk streaming). Cloud streaming is production-preferred with server-side VAD + endpointing; the final text undergoes clean_asr_text post-processing (Overlap Stitcher → Hallucination Filter → Length Guard).

Barge-in Controller

TTS playback state reverse-gates VAD via /tts/playback_active, forming a closed-loop dialogue guard: when TTS is speaking, VAD is suppressed to prevent the robot from hearing its own voice. Users can interrupt at any time — barge-in detection immediately stops TTS and re-enables the pipeline.

Key Result: Far-field Silero hit rate 0% → 92%; VAD discrimination: white noise prob < 0.01, real speech prob > 0.99; partial ASR latency 150–300 ms.

VLM bbox → SAM3 Mask (Letterbox Alignment)

Key fix: unified scale ratio from separate 0.7875/1.4 to min(size/W, size/H), preserving 16:9 aspect ratio. The API passes original-image pixel coordinates; a layout-aware inverse function maps them into letterbox-internal coordinates. This prevents ViT from seeing targets stretched 1.78× vertically. Combined with CLIP English text joint grounding (HAR_SAM3_BOX_TEXT_EMPTY=0), small-object mask success jumped from 9% to ≥90% (44 green-apple trials).

5-Layer Semantic Defense Chain

Layer Mechanism Purpose
1 Whitelist longest-substring recovery Recovers objects from ASR sliding-window noise
2 ASR color backfill into prompt Fixes NLU dropping color attributes
3 Hallucinated location stripping Removes phantom placement locations from LLM
4 Translation sentinel blacklist Breaks dictionary-miss infinite loops
5 VLM-SAM3 IoU / center-distance validation Catches VLM-SAM3 spatial contradictions before cache write
Key Results: SAM3 mask success (small targets): 9% → ≥90%; VLM bbox TTFB saved 300–500 ms; color/brand routing error rate → 0.

Grasp Sub-FSM (5 States)

stateDiagram-v2
    [*] --> idle
    idle --> executing: GraspFeedback.STARTED
    executing --> verifying: EXEC_DONE
    verifying --> idle: pass / VERIFY_SKIP
    verifying --> retrying: fail
    retrying --> executing: re-EXEC_GRASP
    retrying --> aborting: max retries

    aborting --> idle: retract + TTS

The main gate (median < 0.055m) is mandatory — if the distribution center is too far, the object is clearly not in hand. Three auxiliary voters (P25, frac<3cm, frac<5cm) each capture different contact evidence; any one passing suffices. SAM3 runs batch=2 to output both object and hand masks simultaneously, avoiding cross-frame misalignment.

P0 Bug Case Study

Symptom: grasping “apple” repeatedly retried 3+ times, always failing. Root cause: _bg_write_shm() success path only logged but never set shm_write_ok["ok"]=True. This single missing flag triggered a 7-step cascading error chain: flag False → fallback path → stale parser state → VLM sees old frame → SAM3 gets wrong bbox → mask too small (27/65/93 px) → depth_meta=None → fail-fast retry. Fix: one line — add shm_write_ok["ok"] = True after successful os.replace().
